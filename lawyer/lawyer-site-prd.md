# PRD: Сайт частного юриста / нотариуса

## Что делаем
Сайт-визитка для частнопрактикующего юриста или нотариуса. Цель сайта одна: превратить
посетителя, у которого есть юридический вопрос, в заявку на консультацию. Всё остальное
подчинено этой цели.

## Кто пользователь
- **Потенциальный клиент** — человек с конкретной проблемой (развод, спор с соседом,
  оформление наследства, регистрация фирмы). Он нервничает, не разбирается в праве,
  боится ошибиться в выборе юриста. Ему нужна уверенность, что юрист компетентен
  и не обманет.
- **Сам юрист** — хочет получать заявки, не отвечать на одни и те же вопросы вручную
  и выглядеть профессиональнее конкурентов.

## Почему это работает (контекст для Cursor)
По данным исследований 2026 года: 96% людей ищут юриста через поисковик, 67% трафика
на юридические сайты приходит с мобильных телефонов, 75% доверия к сайту формируется
дизайном, а 40% мобильных пользователей звонят прямо с главной страницы. Сайт должен
работать как клиентский менеджер — ответить на вопрос "можно ли вам доверять" за первые
10 секунд просмотра.

## Структура сайта (все секции на одной странице, single page)

### 1. Шапка (header) — фиксированная при скролле
- Логотип / название практики слева
- Навигация: Услуги / О юристе / Как работаем / Контакты
- Кнопка "Записаться" — всегда видна, выделена цветом
- Телефон кликабельный (`tel:` ссылка) — обязательно, 40% звонят с мобильного прямо
  отсюда

### 2. Hero-блок (первый экран)
- Короткий заголовок: что делает юрист + для кого + главное преимущество.
  Пример: "Защищаю права граждан и бизнеса в [город]. Первая консультация бесплатно."
  НЕ писать "Мы — команда профессионалов с многолетним опытом" — это шаблон.
- Подзаголовок: 1–2 предложения, от лица клиента, про его боль, не про регалии юриста
- Две кнопки: "Записаться на консультацию" (главная) + "Узнать об услугах" (вторичная)
- Три цифры-факта под кнопками (пример: "12 лет практики / 340 дел / 94% успешных")
  — реальные данные от клиента-юриста, не придуманные

### 3. Услуги (practice areas)
- Сетка карточек: 4–6 направлений (пример: гражданское право / семейное / корпоративное /
  нотариальные услуги / наследство / трудовые споры)
- Каждая карточка: название + 2–3 строки объяснения простым языком (не юридическим)
  + примеры ситуаций ("развод и раздел имущества", "взыскание долга", "оформление
  наследства")
- Каждая карточка кликабельна — разворачивает детальное описание (accordion) или
  открывает модальное окно с подробностями

### 4. О юристе (bio)
Это самая важная секция по данным исследований: люди нанимают человека, не компанию.
- Фото юриста (реальное, не сток) — большое, качественное
- Имя + специализация + стаж
- Короткий текст "от первого лица" — не официальная биография, а живой рассказ
  о подходе к работе
- Список образования, членства в коллегиях, ключевых достижений — строго фактически,
  без превосходных степеней
- Кнопка "Задать вопрос" — прямая ссылка к форме записи

### 5. Как работаем (process)
- 4 шага: Консультация → Анализ → Ведение дела → Результат
- Каждый шаг: название + 2–3 строки что происходит и что получает клиент
- Упор на прозрачность: клиент должен понимать, что не будет неприятных сюрпризов

### 6. Форма записи на консультацию
Главная конверсионная точка — размещать и вверху (в hero) и отдельной секцией внизу.
Поля формы (минимально необходимые):
- Имя (текст, обязательно)
- Телефон или Telegram (текст, обязательно)
- Область права (select: список из направлений юриста + "другое")
- Краткое описание ситуации (textarea, необязательно — снижает барьер входа)
- Кнопка отправки

После отправки:
- Показать сообщение "Заявка принята. Свяжемся в течение 2 часов в рабочее время."
- Отправить уведомление юристу (на email и/или в Telegram-бот — см. Этап 2)

Важно: НЕ добавлять лишние поля (дата рождения, паспорт и т.п.) — каждое лишнее поле
снижает конверсию. Минимум полей → максимум заявок.

### 7. Отзывы (social proof)
- 3–5 отзывов от реальных клиентов: имя (можно инициалы), ситуация, результат
- Формат: цитата + короткий контекст ("спор о наследстве, 2024")
- Без фотографий клиентов — нотариальная тайна, конфиденциальность

### 8. Футер
- Контакты: телефон (кликабельный), email, Telegram, адрес + карта (embed Google Maps
  или 2ГИС)
- Часы работы
- Краткий список услуг со ссылками на секции страницы
- Юридический дисклеймер: "Информация на сайте носит ознакомительный характер
  и не является юридической консультацией."

## Дизайн-система (передать Cursor в виде CSS переменных)

Концепция: строгий, дорогой, современный. Не казённый советский стиль, не кричащий
агентский дизайн. Ориентир: высококлассная европейская юридическая фирма.

```css
:root {
  /* Палитра: глубокий тёмно-синий + тёплый слоновая кость + золото */
  --color-navy:       #0D1B2A;   /* фон hero, шапки, футера */
  --color-navy-mid:   #1A2E45;   /* вторичные тёмные блоки */
  --color-gold:       #B8973A;   /* акцент: кнопки, линии, иконки */
  --color-gold-light: #D4AE56;   /* текст на тёмном фоне */
  --color-ivory:      #F7F5F0;   /* основной фон страницы */
  --color-ivory-dark: #EDE9E0;   /* фон чередующихся секций */
  --color-text:       #0D1B2A;   /* основной текст */
  --color-muted:      #5C6B7A;   /* вспомогательный текст */
  --color-line:       #D4CFC4;   /* разделители, рамки */

  /* Типографика */
  --font-display: 'Playfair Display', Georgia, serif; /* заголовки */
  --font-body:    system-ui, -apple-system, sans-serif; /* текст, UI */

  /* Шрифты из Google Fonts (добавить в <head>):
     <link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,400;0,600;1,400&display=swap" rel="stylesheet"> */

  /* Размеры */
  --radius: 0px;          /* NO rounded corners — строгий прямоугольный стиль */
  --max-width: 1100px;    /* максимальная ширина контейнера */
}
```

### Типографика
- **Заголовки H1–H3**: Playfair Display, weight 400 (не bold — серьёзность без крика)
- **Тело, формы, кнопки**: system-ui (sans-serif)
- **Кнопки**: все заглавные буквы, letter-spacing 0.1em, size 12–13px
- **Акцент-метки (eyebrow text)**: 11px, letter-spacing 0.2em, uppercase, цвет gold

### Компоненты
- **Кнопка Primary**: фон `--color-gold`, текст `--color-navy`, 0 скруглений
- **Кнопка Secondary/Outline**: прозрачный фон, рамка 1px белая (на тёмном) или navy
  (на светлом)
- **Карточки услуг**: белый фон, рамка 1px `--color-line`, 0 скруглений,
  золотая линия-акцент 2px сверху или слева
- **Разделитель секции (eyebrow)**: маленький текст + горизонтальная линия после него

### Ключевая деталь (signature element)
Символ параграфа **§** — декоративно используется в логотипе и как крупный фоновый
элемент в hero (opacity 0.03–0.05). Это единственный "фокус" дизайна — всё остальное
строгое и чистое.

## Технический стек

- **HTML + CSS + минимальный vanilla JS** — без фреймворков, сайт должен быть
  максимально быстрым (юридические сайты: 50% уходят если загрузка >3 сек)
- Форма: отправка через `fetch` POST-запрос на простой бэкенд endpoint
- Бэкенд (минимальный): Python + Flask или Node + Express — принимает форму,
  сохраняет в файл/SQLite, отправляет email-уведомление (через SMTP/SendGrid)
- Хранение заявок: SQLite (не продакшен-база данных юриста)
- Деплой: VPS (Timeweb/Beget) или Netlify (если обойтись без бэкенда на MVP)

### Адаптивность (обязательно)
- Mobile-first: сначала верстать мобильную версию, потом расширять на десктоп
- Контрольные точки: 375px (small phone), 768px (tablet), 1100px (desktop)
- Телефон в шапке: всегда кликабелен (`<a href="tel:+373XXXXXXX">`)
- Форма: все поля на всю ширину на мобильных

## Что НЕ делаем (MVP)

- НЕ делаем онлайн-оплату услуг юриста
- НЕ делаем личный кабинет клиента
- НЕ интегрируемся с судебными базами данных или реестрами
- НЕ делаем мультиязычность (один язык — тот, на котором говорит целевая аудитория)
- НЕ делаем блог/статьи — это Этап 2
- НЕ используем стоковые фото юридической тематики (молоток судьи, Фемида, колонны)
  — они убивают доверие, выглядят дёшево

## Этап 2 (после сдачи MVP, по договорённости)

- Telegram-бот уведомлений: новая заявка с сайта → сообщение юристу в Telegram
  с именем, контактом и описанием ситуации
- Страница FAQ: 10–15 частых вопросов клиентов простым языком — повышает SEO
  и снижает нагрузку на юриста
- Раздел с делами/кейсами: анонимизированные примеры выигранных дел — строит доверие
- Счётчик посещений и форм через Яндекс.Метрика или Google Analytics

## Контент, который нужно получить от клиента-юриста ДО начала разработки

Без этого разработку начинать нельзя — без реального контента сайт не выглядит
профессионально:

- [ ] Фото юриста (минимум 1, качественное, деловой стиль)
- [ ] ФИО, специализация, стаж
- [ ] Список направлений работы (4–6 пунктов)
- [ ] Текст "о себе" (можно записать голосом — переведём в текст)
- [ ] Образование и членство в коллегиях
- [ ] 3–5 отзывов от клиентов (можно скопировать из Google/2ГИС)
- [ ] Контакты: телефон, email, Telegram, адрес
- [ ] Часы работы
- [ ] Город и регион практики

## Acceptance criteria (как проверить, что готово)

- [ ] Сайт открывается на мобильном телефоне и выглядит нормально
- [ ] Кнопка "Записаться" в шапке работает (скроллит к форме)
- [ ] Телефон в шапке кликабелен на телефоне (открывает звонок)
- [ ] Форма: нельзя отправить без обязательных полей
- [ ] Форма: после отправки появляется сообщение подтверждения
- [ ] Заявка с формы реально приходит юристу (проверить на тестовом email)
- [ ] Страница загружается быстро (проверить через Google PageSpeed Insights — цель 80+)
- [ ] Ни одной стоковой фото с молотком судьи или Фемидой
